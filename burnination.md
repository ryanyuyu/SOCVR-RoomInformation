## Burnination Backlog

Burnination is the [process of removing a tag](http://meta.stackoverflow.com/questions/324070/what-is-the-process-for-burninating-tags) from Stack Overflow. This is done by moderating the questions bearing that specific tag 
until no questions have that tag anymore. The goal is to keep the value and get rid of the garbage.

### Burninate Requests and candidates

[Search](http://meta.stackoverflow.com/search?tab=votes&pagesize=50&q=%5bburninate-request%5d%20closed%3ano%20-%5bstatus-*%5d%20hasaccepted%3ano) for burninate requests that are not closed and have no status.  
You can go over these post and comment or edit to improve their accurateness or answer if you feel burnination of the tag is not warranted.

[This are the requests that passed the minimal requirement](http://meta.stackoverflow.com/search?q=%5Bburninate-request%5D+closed%3Ano+-%5Bstatus-*%5D+hasaccepted%3Ano+score%3A20)  
Those posts will be up for burnination. These posts need to be up to date and into shape so the burniation can start. Edit or comment if things need to be updated. When the tag has less then 50 questions on main consider taking it on yourself.


[This is the SEDE Query we use to come up with the order in which the SOCVR handles burninate requests](http://data.stackexchange.com/meta.stackoverflow/query/493425/burninate-priority-list)   
You find the list below. The query forms our backlog. You can help out by focussing on the top posts and verifying their data and help by suggesting re-tag solutions, moderation hints. Those will be moved to the progress answer once the burnination is taken on by the SOCVR. If you feel another room is better equipped to carry out the burniation, please bring it up there.
